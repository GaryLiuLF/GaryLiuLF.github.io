用 Hexo + Github 搭建自己的博客

发表于 2019-01-17   |   更新于 2019-01-18 | 分类于 Hexo | 阅读次数:


前言：个人安装博客的环境是 macOS 操作系统，因个人已安装好博客，其中有些步骤的终端界面不能逐一截图

总述

搭建的过程分为以下几步：

  1. 安装 Node.js
  2. 安装 Git
  3. 安装 Hexo 博客框架
  4. 本地测试运行
  5. 本地部署到 GitHub 上
  6. 其他（优化页面，第一篇博文，设置 categories，tags）

安装 Node.js

  1. 打开终端命令行，输入node -v npm -v，验证是否已安装

    终端输出以上的内容，说明已安装。
  2. 若没安装，Node.js 下载地址

安装 Git

  1. 输入git --version，验证是否已安装

    终端输出以上的内容，说明已安装。
  2. 如果没安装，Git 下载地址

安装 Hexo 博客框架

  1. 输入 Hexo 的安装命令npm install -g hexo-cli
  2. 若终端出现以下内容，说明安装报错了
  3. 出现这问题，输入sudo npm install --unsafe-perm --verbose -g hexo命令行和开机密码继续完成安装
  4. 最后输入hexo version，若看到版本号，恭喜你，已安装成功！
  5. 依次执行以下命令行

    hexo i blog // init 的缩写 blog是项目名，项目名可以任意写，本人是blog
    cd blog // 切换到站点根目录
    hexo g // 完整命令为hexo generate,用于生成静态文件
    hexo s // 完整命令为hexo server,用于启动服务器，主要用来本地预览

  6. 打开浏览器输入localhost:4000

    当你看到这种页面，再次恭喜你，你又成功了，这就是 hexo 默认的博客主题，是时候书写你的文采了
  7. 很显然这默认主题，我们是不喜欢的，谁让我们如此任性呢，那就继续盘它，刚好，github上提供了很多样式的主题，而喜欢简洁明了的我，选择了 next 主题，这主题也是目前很流行的主题，我们说干就干
  8. 在站点根目录下输入git clone https://github.com/iissnan/hexo-theme-next themes/next
  9. 完成后，找到站点目录，打开站点配置文件
  10. 打开配置文件，找到 theme 字段，并将其值改为next
  11. 在终端输入以下命令行

    hexo clean // 清除缓存
    hexo g // 重新生成代码
    hexo s // 部署到本地
    // 再在浏览器打开访问 localhost:4000 看看页面

  12. next 主题有四种选择，本人选择的是 Mist，你也可以每个都试试，选出自己最爱的那个，开心就好
  13. 然后再重新输入以下命令行，测试下效果

    hexo clean // 清除缓存
    hexo g // 重新生成代码
    hexo s // 部署到本地

  14. 当然，以上的步骤，只是冰山的一小角，你可以进行很多的自定义设置甚至修改源码，来个私人订制，自己喜欢的博客样式，而且NexT官网也提供了官方文档
    官方传送门 >>

本地部署到 GitHub 上

废话不多说，继续盘它

  1. 登录自己的 github 账号，若没有，注册去
    官方传送门 >>
  2. 创建一个新仓库
  3. 按照指定格式输入仓库名
  4. 点击 Settings，创建一个 page
    本人看了其他的牛人搭建博客到这步骤，说要找到Overwrite site，当时本人确实没找到，就很难受，但也没事，跳过这步骤，继续盘它（如果有人知道的，指点我一下，三克油^_^），在此本人就写写自己的后续步骤
  5. 选择一个主题，其实没什么用，反正后续操作也是要被覆盖的
  6. 这时候你用之前创建的仓库名用户名.github.io，在浏览器中访问下这域名

    若能成功打开页面，再一次恭喜你，你又成功了一大步，接下来就是将 Hexo 的代码部署到 GitHub 上
  7. 打开 hexo 站点目录，找到站点配置文件，修改deploy

    修改之后，可要记得保存

    切记，配置文件中，冒号的后面一定一定一定要有一个空格！！！（重点的事情说三遍）

  8. 先安装个插件，再部署一下就ok了，输入以下命令行

    npm install hexo-deployer-git –save // 安装个插件
    hexo d // 部署网站的命令，也就是发布下

  9. 再次用新建的仓库名用户名.github.io，在浏览器中访问下这域名

    若能打开，就说明你真的成功了，是不是 so easy，就很 nice
  10. 发布第一个博客
    打开终端，在根目录下输入以下命令行

    hexo n “proName” // hexo new “proName”
    // 在根目录下的 source/_posts 下就有个 proNamed.md的文件
    // 在文件中用 makedown 语法编写就行
    编写之后保存，在根目录下，输入以下命令行
    hexo g // 生成静态页面
    hexo d // 发布
    // 后期本人直接是 hexo g -d 操作

后记

用 Hexo + GitHub 搭建个人博客，步骤其实很简单也很基础的，本人博客主题也是用 NexT ，简洁明了，本人喜欢的风格，博客页面也是在不断优化中。如果你还有更多的想法，可以去 NexT 官方文档看看，逐个修改设置下，也可以与我一同分享。
